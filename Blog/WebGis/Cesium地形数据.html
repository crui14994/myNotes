Cesium 地形数据格式 ​

Cesium 1.120 强制要求地形数据必须是Quantized Mesh v1.1格式; 且 layer.json 必须存在且路径正确！

  • 性能提升 ‌：v1.1 格式通过更高效的编码方式减少了数据存储大小，提升了地形数据的加载和渲染速度 ‌

  • 功能增强 ‌：特别引入了对地形特征和元数据的支持，允许开发者在地形数据中嵌入更丰富的信息，如建筑轮廓、道路网络等地理要素 ‌!

  • 兼容性改进 ‌：更好地与 3D Tiles 1.1 标准集成。

Quantized Mesh 格式 ​

注意

  • 旧版 Quantized Mesh v1.0（不包含法线）
  • Quantized Mesh v1.1 格式（支持法线/边缘信息）

Quantized Mesh v1.1 与 v1.0 的主要区别

特性v1.0 版本v1.1 版本
‌ 数据结构简单的三角网格支持特征和元数据扩展
‌ ‌ 编码方式基本量化压缩更高效的量化算法
‌ 文件大小相对较大减少约 20-30%
‌ 功能支持基本地形渲染支持高级地形特征
‌ 兼容性仅限基础地形与 3D Tiles 1.1 完全兼容

Windows 系统 Docker 操作详细步骤 ​
  1. 拉取镜像
bash
# tumgis/ctb-quantized-mesh - 支持最新的量化网格格式
# homme/cesium-terrain-builder - 经典稳定版本
docker pull tumgis/ctb-quantized-mesh

# 如果报错正在安装: 适用于 Linux 的 Windows 子系统 灾难性故障 必须运行 wsl --update 命令来更新 WSL
# 使用 --web-download 参数可以从 GitHub 等在线源直接下载更新，通常能获得更快的下载速度并避免网络问题
wsl --update --web-download

  1. 地形切片生成流程
bash
# 步骤1：进入容器环境
docker exec -it ctb /bin/bash

# 步骤2：创建GDAL虚拟数据集（可选）
gdalbuildvrt tiles.vrt /data/84wgs.tif

# 步骤3：生成Cesium地形文件
ctb-tile -f Mesh -C -N -o /data/output tiles.vrt

# 步骤4：创建图层描述文件
ctb-tile -f Mesh -C -N -o /data/output tiles.vrt -l

# 1. 确保使用最新版镜像
docker pull tumgis/ctb-quantized-mesh

# 2. 直接运行完整命令（无需进入容器）
docker run --rm -v "e:/docker/terrain:/data" tumgis/ctb-quantized-mesh \
  ctb-tile -f Mesh -C -N -s 18 -o /data/output /data/84wgs.tif

# 3. 生成layer.json（关键步骤）
docker run --rm -v "e:/docker/terrain:/data" tumgis/ctb-quantized-mesh \
  ctb-tile -f Mesh -C -N -o /data/output /data/84wgs.tif -l
