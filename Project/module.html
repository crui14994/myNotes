概述 ​

子模块位于项目/src/modules/文件目录下

以basicDataSource模块为例。本指南将介绍如何创建、配置和打包一个新子模块。

  • 子模块之间的路由是完全与主模块独立的，但是共用一个路由文件。
  • 子模块与主模块共享 store，通过设置持久化存储方案来实现数据共享。
  • 打包的时候可以通过配置文件单独打包一个或多个模块。

注意事项 ​

  • 确保每个子模块都有独立的命名空间
  • 路由配置应避免与其他模块冲突
  • 使用懒加载提高性能
  • 维护良好的目录结构和文件命名规范
  • 如需打包部分子模块只需在vite.config.js入口中注释掉不需要打包的模块！

:::

1. 拷贝一个子模块重新命名为 basicDataSource；目录结构大致为。 ​
/src/modules/basicDataSource
├── api                    # API接口定义
├── assets                 # 静态资源文件
├── components             # 可复用组件
└── views                  # 页面视图
    ├── argBasicSource     # 示例视图
    └── ...               # 其他视图
├── App.vue                # 模块入口组件
└── main.js                # 模块入口文件

4.在 src/utils/moduleRouteLoadView.js 中对请求到的路由进行过滤组装，过滤掉不需要的子模块路由。 ​
点我查看代码
js
...
// 匹配基础资源库views里面所有的.vue文件
const basicDataSourceModules = import.meta.glob("@basicDataSource/views/**/*.vue");
function basicDataSourceRouteLoader(view, e) {
  // 路由懒加载e)
  switch (e) {
    case "argBasicSource":
      return basicDataSourceModules[`/src/modules/basicDataSource/views/argBasicSource/${view}/index.vue`];
    case "plantingResources":
      return basicDataSourceModules[`/src/modules/basicDataSource/views/plantingResources/${view}/index.vue`];
    case "monitorResource":
      return basicDataSourceModules[`/src/modules/basicDataSource/views/monitorResource/${view}/index.vue`];
    case "service":
      return basicDataSourceModules[`/src/modules/basicDataSource/views/service/${view}/index.vue`];
    default:
      return basicDataSourceModules[`/src/modules/basicDataSource/views/${view}/index.vue`];
  }
}

...
export default {
  defaultRouteLoader,
  //注意命名为模块名称
  basicDataSource: basicDataSourceRouteLoader,
  ...
};
